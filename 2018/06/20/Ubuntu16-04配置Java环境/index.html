Ubuntu16.04配置Java环境

发表于 2018-06-20 | 分类于 学习笔记
字数统计: 545 | 阅读时长 ≈ 2 分钟

配环境给我配的真的是要死要活的,win10很轻松,很顺利的就配好了,Ubuntu折腾了老半天,现在我把我配置的过程记录下来,省得以后再配置的时候不好弄.

1. 官网下载jdk

下载网址:http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html,我下载的是jdk-8u171-linux-x64.tar.gz.32位系统选择x86,64位选择x86.

2. 安装jdk

依次执行以下命令

sudo mkdir /usr/lib/Java    //创建一个新的文件夹
cd 下载
sudo tar zxvf jdk-8u171-linux-x64.tar.gz -C /usr/lib/Java    //将下载的压缩包解压到该文件夹中

3. 配置环境变量

命令行打开gedit编辑/etc/profile 

sudo gedit /etc/profile

添加以下代码到profile文件末尾,其中版本号与文件路径视自己的情况而定.

export JAVA_HOME=/usr/lib/Java/jdk1.8.0_171
export JRE_HOME=$JAVA_HOME/jre
export CLASSPATH=.:$CLASSPATH:$JAVA_HOME/lib:$JRE_HOME/lib
export PATH=$PATH:$JAVA_HOME/bin:$JRE_HOME/bin

使得/etc/profile文件生效

source /etc/profile

用以下命令查看是否成功

java -version

完成以上配置后,我发现了一个问题,环境配置只有在source /etc/profile命令执行后才生效,这样就很不方便.于是,我又修改了其他的环境变量.

sudo gedit /etc/environment    //打开环境变量的配置文件

改为如下内容

PATH="/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games:$JAVA_HOME/bin"
export CLASSPATH=.:$JAVA_HOME/lib:$JAVA_HOME/jre/lib
export JAVA_HOME=/usr/lib/Java/jdk1.8.0_171

其实就是在path路径下添加:$JAVA_HOME/bin(添加在双引号内),在后两行再添加上面所示的内容,然后执行以下命令使配置立即生效

source /etc/environment

话虽然是这么说,但是我其实是在重启机器后才生效的.

另外,用vim写java的时候,先进去你存放java文件的文件夹,然后再打开或新建java文件.主要原因是因为java命令后不能加路径,只能直接在当前文件夹下用java命令来运行class文件.

好了.这样我们的配置就大功告成了,这下就可以舒舒服服的学Java了.
