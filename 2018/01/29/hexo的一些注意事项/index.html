hexo的一些注意事项

发表于 2018-01-29 | 分类于 学习笔记

  1. 头像问题

    在hexo的next主题中，想要修改头像，应该先找到你安装next的文件夹，比如我的是~/blog/themes/next。在该文件夹中找到_config.yml文件，改动下面这行代码。

    avatar: /images/avatar.jpg

    这是我改完之后的代码，然后就可以将你想要设定为头像的图片重命名为avatar.jpg，
    将其放入next文件夹下的source/images/中就可以了。

  2. 插入图片问题

    在写文章插入本地图片的时候，预览的时候没问题，但是部署到网站上之后，并不显示图片。具体原因我也不太清楚，反正markdown的语法并没有起作用。解决方法如下：
    这一次进入blog目录（就是你安装hexo的目录），打开_config.yml文件，改动如下的代码

    post_asset_folder: true

    然后在blog目录下运行如下命令安装插件

    npm install hexo-asset-image --save

    安装完这个插件，在你用hexo new 命令创建一篇文章之后，会在~/blog/source/_posts下生成一个跟文章同名的文件夹（也就是和文章在同一文件夹）。你只需要将图片放入该文件夹中，然后用如下的插件标签在文章中插入图片即可。

    {% asset_img 图片名.后缀名 %}

  3. 代码块高亮问题

    引用代码块无法高亮的，可以用“ ```”后面加上语言，如cpp，然后下一行贴代码块，代码块结束后，在下一行加上“ ``` ”，注意：此处为反引号，Tab键上面的那个。

markdown中转义使用反斜线\，而不是斜线/

# hexo
